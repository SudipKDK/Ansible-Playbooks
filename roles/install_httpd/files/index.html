Quick Intro to Ansible

A brief, beginner-friendly overview and tiny examples to get you started.

What is Ansible?

Ansible is an open-source automation tool used for configuration management, application deployment, task automation, and orchestration. It uses simple, human-readable YAML files called playbooks to describe automation tasks.

Why use Ansible?

  • Agentless — communicates over SSH (no agent installation required).
  • Declarative — describe the desired state; Ansible makes it so.
  • Idempotent — safe to run multiple times without unintended side effects.
  • Fast to learn — playbooks use simple YAML and many built-in modules.

Small example — inventory and playbook

Inventory (hosts file):

# inventory/hosts
[web]
web1.example.com
web2.example.com

[db]
db1.example.com

Playbook: install Nginx and start it

---
- name: Install and start Nginx on web hosts
  hosts: web
  become: yes
  tasks:
    - name: Ensure Nginx is installed
      package:
        name: nginx
        state: present

    - name: Ensure Nginx service is running
      service:
        name: nginx
        state: started
        enabled: true

Run the playbook:

ansible-playbook -i inventory/hosts playbooks/install-nginx.yml

Common Commands

  • ansible all -m ping -i inventory/hosts — check connectivity.
  • ansible-playbook playbook.yml -i inventory/hosts --check — run in dry-run mode.
  • ansible-vault encrypt secrets.yml — encrypt sensitive data.

Tips & Best Practices

  • Keep inventory separate from playbooks.
  • Use roles for reusable, organized code (use ansible-galaxy init).
  • Protect secrets with Ansible Vault.
  • Test with --check and use CI to lint playbooks (ansible-lint).

Further Learning

Try creating a simple role that installs a package and deploys a configuration file. When you're ready, explore dynamic inventories, custom modules, and integrating Ansible with CI/CD pipelines.
